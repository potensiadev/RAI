 🔴 FAANG PM Critical Analysis Report

  Executive Summary

  RAI는 기술적으로 잘 설계된 플랫폼이지만, 사용자 관점에서 치명적인 UX 문제들이 존재합니다. 이는 사용자 이탈과 NPS 하락으로 직결됩니다.

  ---
  🚨 CRITICAL: 치명적 단점 (Day 1 이탈 유발)

  1. 검색이 안 된다고 느끼는 사용자 경험

  문제: "React개발자" 검색 시 결과 없음
  원인: 한영 붙여쓰기 쿼리 미지원 (이제 수정됨)
  영향: 95%의 헤드헌터가 이 패턴으로 검색
  PM 관점: 검색은 서비스의 핵심입니다. 검색이 안 되면 서비스가 없는 것과 같습니다.

  2. 검색 결과에 대한 신뢰 부재

  문제: 왜 이 후보자가 매칭됐는지 설명 없음
  현재: 그냥 리스트만 보여줌
  기대: "React 6년 경력, 금융업 경험" 하이라이트
  PM 관점: Google도 검색 결과에 bold 처리를 합니다. 사용자는 AI를 신뢰하지 않습니다.

  3. 빈 스킬 태그가 UI에 표시됨

  문제: 스킬 필터에 빈 태그 "　" 표시
  원인: skills 배열에 null/empty string 포함
  영향: "이 서비스 제대로 만든 거 맞아?" 신뢰도 하락

  4. 모바일에서 데이터 무한 로딩

  문제: 지하철 등 불안정 네트워크에서 리스트 계속 깜빡임
  원인: 네트워크 재연결 시 debounce 없음
  영향: 배터리 소모 + 사용자 짜증

  ---
  🟠 HIGH: 주요 개선사항 (Weekly Active User 감소 유발)

  5. "이미 본 후보자" 필터 없음

  시나리오: 100명 검색 → 20명 검토 → 다음 날 같은 검색
  문제: 이미 본 20명이 또 나옴
  기대: "새로운 후보자만 보기" 토글

  6. 유사 이름 검색 민감도 조절 불가

  현재: pg_trgm 기본 threshold (0.3) 고정
  문제: "김민수" 검색 시 "김민" "김수민" 다 나옴
  기대: 슬라이더로 엄격/느슨 조절

  7. 검색어 분리 결과 미표시 (이제 수정됨)

  Before: "C++개발자" 검색 → 그냥 결과만 보여줌
  After: 검색어: [C++] [개발자] 태그 표시 ✅

  8. 대량 선택 기능 없음

  시나리오: 50명 중 10명 선택해서 한번에 export
  현재: 1명씩 클릭 → export → 뒤로가기 → 반복
  경쟁사: Greenhouse, Lever 모두 bulk action 지원

  9. 저장된 검색의 변경 이력 없음

  문제: 저장된 검색 수정 후 "어떻게 바꿨더라?"
  현재: 버전 관리 없음
  기대: 최소 last modified 표시

  ---
  🟡 MEDIUM: 잠재적 불만 (Churn Risk 증가)

  10. 검색 히스토리 탭 간 동기화 안 됨

  문제: 탭1에서 검색 → 탭2에서 히스토리 안 보임
  원인: localStorage만 사용, 서버 동기화 없음

  11. 업로드 진행률 부정확

  문제: 80%에서 갑자기 100%로 점프
  원인: 파싱/분석 단계 granularity 부족
  기대: "파싱 중... → AI 분석 중... → 저장 중..."

  12. 에러 메시지 기술적

  Before: "RATE_LIMIT_EXCEEDED"
  After: "잠시 후 다시 시도해주세요 (분당 30회 제한)"

  13. 중복 후보자 자동 감지/병합 없음

  시나리오: 같은 사람 이력서 2개 업로드
  현재: 2개 별도 후보자로 등록
  기대: "기존 후보자와 유사합니다. 병합하시겠습니까?"

  14. Position 매칭 결과 푸시 없음

  문제: Position에 맞는 새 후보자 등록되어도 알림 없음
  기대: "김철수님이 Frontend Lead 포지션과 85% 매칭됩니다"

  15. 스킬 동의어 확장 결과 불투명

  문제: "React" 검색 시 React Native도 매칭되는지 모름
  기대: "React, React Native, ReactJS 포함" 표시

  결론

  RAI는 기술적 기반이 우수하지만 (AI 2중 검증, 암호화, Hybrid Search), 사용자 경험 디테일에서 미흡합니다.

  가장 시급한 것은:
  1. 검색 결과에 대한 투명성 - 왜 이 후보자인가
  2. 데이터 품질 - 빈 태그, 중복 제거
  3. 모바일 안정성 - 네트워크 플래핑 대응

  이 3가지만 해결해도 NPS +20 예상됩니다.
