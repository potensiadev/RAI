## Part 1: 검색 UX 개선 필요사항

### 1.1 검색어 자동 교정/제안 부재 [HIGH]

**현재 상태**:
- 사용자가 "리엑트" (오타) 입력 → 결과 없음
- 사용자가 "React개발자" (붙여서) 입력 → 결과 없음

**사용자 시나리오**:
```
1. 헤드헌터가 빠르게 "리엑트 시니어" 검색
2. 결과 0건 표시
3. 오타인지, 해당 인재가 없는지 구분 불가
4. 다시 "React 시니어" 검색 → 결과 나옴
5. 사용자 짜증 + 시간 낭비
```

**권장 개선안**:
```
[검색 결과 0건일 때]
"리엑트"에 대한 결과가 없습니다.
혹시 "React"를 찾으시나요? [바로 검색]
```

**비즈니스 가치**:
- 검색 실패율 감소
- 사용자 이탈 방지
- "이 시스템 똑똑하네" 인상

---

### 1.2 검색 결과 매칭 이유 미표시 [HIGH]

**현재 상태**:
- "React 개발자" 검색 → 결과 표시
- 왜 이 후보자가 나왔는지 알 수 없음

**사용자 시나리오**:
```
1. "React 시니어 핀테크" 검색
2. 후보자 목록 표시
3. 어떤 후보자가 React 때문에 나왔는지, 핀테크 때문인지 불명확
4. 일일이 프로필 열어서 확인해야 함
```

**권장 개선안**:
```
[후보자 카드]
홍길동 - Senior Frontend Developer
━━━━━━━━━━━━━━━━━━━━━━━━━━━
Match:
✓ React (스킬 일치)
✓ "시니어" → Senior (동의어 매칭)
? 핀테크 (직접 일치 없음, 유사 업종)
```

**비즈니스 가치**:
- 클라이언트 보고서 작성 용이
- 잘못된 후보자 선별 시간 절약
- 시스템 신뢰도 향상

---

### 1.3 동의어 확장 결과 투명성 부족 [MEDIUM]

**현재 상태**:
- "개발자" 검색 → 내부적으로 "developer", "Engineer" 등으로 확장
- 사용자는 이 사실을 모름

**사용자 시나리오**:
```
1. "백엔드 개발자" 검색
2. "Backend Engineer" 포지션 후보자도 결과에 포함
3. 사용자: "나는 '개발자' 검색했는데 왜 Engineer가 나오지?"
4. 시스템 불신 또는 혼란
```

**권장 개선안**:
```
검색어: "백엔드 개발자"
━━━━━━━━━━━━━━━━━━━━━━━━━━━
확장된 검색어: 백엔드, backend, Backend, BE, 서버
              개발자, developer, Developer, 엔지니어, Engineer
[동의어 확장 끄기]
```

**비즈니스 가치**:
- 검색 결과 이해도 향상
- 사용자 학습 효과 (어떤 단어가 동의어인지 학습)
- 투명한 AI 시스템 이미지

---

### 1.4 부분 매칭 vs 전체 매칭 구분 없음 [MEDIUM]

**현재 상태**:
- "React" 검색 → "React", "ReactJS", "React Native" 모두 매칭
- 어떤 것이 정확히 일치하는지 구분 없음

**사용자 시나리오**:
```
1. 순수 "React" (웹) 개발자만 찾고 싶음
2. "React" 검색
3. React Native 개발자도 함께 나옴
4. 일일이 필터링 필요
```

**권장 개선안**:
```
검색 옵션:
◉ 유사 매칭 (React, React.js, ReactJS 포함)
○ 정확한 매칭 (React만)
```

---

### 1.5 최근 검색어와 자동완성 미연동 [LOW]

**현재 상태**:
- 최근 검색 기록 있음
- 검색창 자동완성에 반영 안 됨

**권장 개선안**:
```
검색창 입력 시:
[최근 검색]
- React 시니어 (2시간 전)
- Python 백엔드 (어제)

[추천 검색어]
- React Native 개발자
- React + TypeScript
```

## Part 2: 데이터 품질 관련

### 2.1 Facet 중복 표시 [HIGH]

**현재 상태**:
- Facet에 "React", "react", "REACT" 가 별도로 표시됨
- 사용자 혼란 + 데이터 품질 의심

**QA 발견 시나리오**:
```
Skills Filter:
☐ React (45)
☐ react (12)
☐ REACT (3)
☐ ReactJS (8)
```

**권장 개선안**:
```
Skills Filter:
☐ React (68)  ← 동의어 그룹화
    includes: react, REACT, ReactJS, React.js
```

**비즈니스 가치**:
- 전문적인 데이터 관리 이미지
- 필터 클릭 수 감소
- 정확한 카운트 제공

---

### 2.2 스킬 정규화 기준 미공개 [MEDIUM]

**현재 상태**:
- 어떤 스킬이 어떤 스킬의 동의어인지 사용자가 알 수 없음
- "JavaScript"와 "ES6"가 같은 것인지 다른 것인지 불명확

**권장 개선안**:
- 동의어 사전 공개 (help/FAQ 페이지)
- 또는 Facet에서 동의어 그룹 표시

---

### 2.3 빈 스킬 데이터 원인 분석 필요 [MEDIUM]

**현재 상태**:
- Hotfix 2로 빈 문자열 필터링 추가
- 하지만 **왜 빈 스킬이 저장되었는지** 원인 미파악

**권장 조치**:
1. 업로드 파이프라인에서 빈 스킬 사전 필터링
2. 기존 데이터 정제 마이그레이션
3. 데이터 품질 모니터링 대시보드

```
[다시 시도] [검색어 단순화하기]
```
